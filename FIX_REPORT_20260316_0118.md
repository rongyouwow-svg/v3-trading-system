## 🔍 根本原因分析

### 问题 1: uvicorn 频繁误报 ❌

**告警内容**:
```
CRITICAL 进程异常：uvicorn 未找到
```

**实际情况**:
```bash
# uvicorn 一直正常运行
admin  30327  0.3  2.7 349720 51880 ?  S  00:58  0:04
/root/.pyenv/versions/3.10.0/bin/python3 -m uvicorn web.dashboard_api:app --host 0.0.0.0 --port 3000

# supervisor 状态正常
quant-web  RUNNING  pid 30327, uptime 0:19:14
```

**根本原因**: **进程名不匹配**

```python
# enhanced_monitor.py 中的配置
EXPECTED_PROCESSES = [
    'uvicorn',  # ❌ 错误：这是进程命令，不是 supervisor 进程名
    'quant-strategy-eth',
    ...
]

# supervisor 实际进程名
quant-web  # ← 正确名称
```

**为什么误报**:
- 监控脚本检查 `process_status['uvicorn']`
- 但 supervisor 状态中只有 `quant-web`
- 找不到 `'uvicorn'` → 触发告警
