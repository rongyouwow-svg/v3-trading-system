### 2.2 核心流程

#### 开仓流程

```
1. 策略模块计算 RSI
   ↓
2. RSI>50 → 发出开仓信号
   ↓
3. 执行引擎接收信号
   ↓
4. 检查仓位限制
   ↓
5. 调用币安 API 开仓
   ↓
6. 获取持仓信息
   ↓
7. 止损管理器创建止损单
   ↓
8. 保存状态到本地
```

#### 止损流程

```
1. 开仓成功 → 获取入场价
   ↓
2. 计算止损价 = 入场价 × (1 - 止损%)
   ↓
3. 检查是否已有止损单（查重）
   ↓
4. 无 → 创建新止损单
   ↓
5. 有 → 更新止损单（分批建仓场景）
   ↓
6. 保存止损单到本地
   ↓
7. 等待价格触发 → 自动平仓
```

## 三、需要重构的模块

### 3.1 策略文件重构（P0）

**当前**:
```python
# strategies/rsi_1min_strategy.py (独立脚本)
class RSIStrategy:
    def run(self):
        while True:
            rsi = self.calculate_rsi()
            if rsi > 50:
                self.open_position()  # ❌ 直接调用 API
```

**重构后**:
```python
# core/strategy/modules/rsi_strategy.py (模块)
class RSIStrategyModule:
    def on_tick(self, market_data):
        rsi = self.calculate_rsi(market_data)
        if rsi > 50:
            return {'action': 'open', 'stop_loss_pct': 0.002}
        return None
```

**改动**:
- ✅ 移除 `run()` 循环
- ✅ 移除 API 调用
- ✅ 只返回信号，不执行
- ✅ 止损配置作为返回值

### 3.2 执行引擎增强（P0）

**当前**:
```python
# core/execution/engine.py
class ExecutionEngine:
    def create_order(...): ...
    def create_stop_loss(...): ...
```

**增强**:
```python
class ExecutionEngine:
    def __init__(self):
        self.strategies = {}  # 策略模块注册表
        self.stop_loss_manager = StopLossManager()
    
    def register_strategy(self, name, module):
        """注册策略模块"""
        self.strategies[name] = module
    
    def run_tick(self):
        """每根 K 线执行"""
        for name, strategy in self.strategies.items():
            signal = strategy.on_tick(self.market_data)
            if signal:
                self.execute_signal(signal)
    
    def execute_signal(self, signal):
        """执行信号"""
        if signal['action'] == 'open':
            # 开仓
            order = self.create_order(...)
            # 同步持仓
            position = self.sync_position(...)
            # 创建止损单
            self.stop_loss_manager.create_stop_loss(...)
```

**设计疏漏**:
- 进程间通信机制未定义
- 止损职责划分不清

**执行偏离**:
- 策略直接调用 API

### 8.2 解决方案

- 代码简单，易维护
