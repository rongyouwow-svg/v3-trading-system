## 🎯 优化详情

### 1. 滑点计算（根据成交量动态调整）

**问题背景：**
传统回测假设交易按收盘价成交，忽略了实际交易中的滑点成本，导致回测结果过于乐观。

**解决方案：**
```python
滑点 = 基础滑点 + 冲击成本
冲击成本 = (仓位 / 当前成交量) × 价格影响系数
```

**流动性等级与基础滑点：**
| 等级 | 日均成交额 | 基础滑点 |
|------|-----------|---------|
| HIGH | ≥ 1 亿 USDT | 0.05% |
| MEDIUM | ≥ 1000 万 USDT | 0.1% |
| LOW | ≥ 100 万 USDT | 0.3% |
| VERY_LOW | < 100 万 USDT | 1% (禁止交易) |

**效果：**
- 更真实地反映交易成本
- 大额交易会自动承担更高滑点
- 避免过度交易低流动性币种

### 2. 仓位限制（单笔 < 成交量 1%）

**问题背景：**
大仓位在低流动性市场会造成显著的价格冲击，传统回测未考虑此限制。

**解决方案：**
```python
最大仓位 = 当前成交量 × 当前价格 × 1%
实际仓位 = min(计划仓位，最大仓位)
```

**效果：**
- 防止大单冲击市场
- 强制分散交易
- 在回测中体现真实可执行的仓位上限

**触发场景示例：**
- 计划仓位：1000 USDT
- 当时成交量：50,000 枚
- 当时价格：10 USDT
- 最大允许仓位：50,000 × 10 × 1% = 5000 USDT
- 结果：✅ 允许执行（1000 < 5000）

### 3. 量价背离检测

**问题背景：**
量价背离是重要的市场反转信号，但传统回测未纳入分析。

**检测逻辑：**

**顶背离（看跌信号）：**
- 价格创新高（上涨 > 2%）
- 成交量萎缩（下降 > 2%）
- 信号：警惕反转

**底背离（看涨信号）：**
- 价格创新低（下跌 > 2%）
- 成交量放大（上涨 > 2%）
- 信号：可能见底

**实现代码：**
```python
def detect_divergence(df, index, window=20):
    # 价格趋势
    price_change = (当前价格 - 期初价格) / 期初价格
    
    # 成交量趋势
    volume_change = (后半段均量 - 前半段均量) / 前半段均量
    
    # 顶背离
    if price_change > 0.02 and volume_change < -0.02:
        return "顶背离 (价格↑成交量↓)"
    
    # 底背离
    if price_change < -0.02 and volume_change > 0.02:
        return "底背离 (价格↓成交量↑)"
```

**效果：**
- 提前识别潜在反转点
- 可作为独立的交易信号
- 帮助规避假突破
