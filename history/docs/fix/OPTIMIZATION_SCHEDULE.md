# 🦞 龙虾王优化节奏说明书

## 分层优化策略 (科学高效)

### 📅 优化周期安排

| 频率 | 时间 | 任务 | 耗时 | 输出 |
|------|------|------|------|------|
| **每日** | 9:00 AM | 轻量更新 | ~5分钟 | 最新数据+看板 |
| **每周** | 周日23:00 | 深度优化 | ~30分钟 | 策略调优+扫描 |
| **每月** | 1号2:00 | 全面回测 | ~2小时 | 8年数据验证 |

---

### 📊 每层做什么

#### 每日 (轻量)
```
动作：
  ✓ 抓取最新行情数据
  ✓ 更新交易看板
  ✓ 生成当日信号
  
不做：
  ✗ 复杂回测 (太耗资源)
  ✗ 策略重训练 (没必要)
```

#### 每周 (深度)
```
动作：
  ✓ 策略参数优化
  ✓ 回测看板更新
  ✓ 山寨币扫描
  ✓ 庄家控盘检测
  
输出：
  - 最佳策略调整建议
  - 下周交易计划
  - 山寨币做空机会清单
```

#### 每月 (全面)
```
动作：
  ✓ 8年历史数据全量回测
  ✓ 自适应策略验证
  ✓ 牛熊周期表现分析
  ✓ 策略库版本迭代
  
输出：
  - 月度绩效报告
  - 策略S/A/B/C评级
  - 下月策略配置建议
```

---

### 🎯 为什么这样安排

**以前 (每小时)：**
- ❌ 数据变化不大，重复计算
- ❌ 8年数据每次都跑，浪费资源
- ❌ 没有沉淀，看不出长期效果

**现在 (分层)：**
- ✅ 日常只看最新，快速响应
- ✅ 周末深度优化，找出规律
- ✅ 月底全面验证，科学决策

---

### 🚨 什么时候打破常规

**立即触发额外优化：**
1. 市场剧烈波动 (BTC单日±15%)
2. 发现重大做空机会 (评分>90)
3. 策略连续亏损 (3笔以上)
4. 用户手动要求

**手动触发命令：**
```bash
# 立即运行日常更新
cd /quant && ./scheduler.sh daily

# 立即运行深度优化
cd /quant && ./scheduler.sh weekly

# 立即运行全面回测
cd /quant && ./scheduler.sh monthly
```

---

### 📈 优化效果追踪

**监控指标：**
- 每日信号准确率 (目标>60%)
- 每周策略胜率变化 (趋势向上)
- 每月总收益率 (跑赢HODL)

**何时升级：**
- 连续2月收益>20% → 升级策略库
- 胜率持续<50% → 重新设计策略
- 发现新市场规律 → 增加策略类型

---

### 💡 本王判断标准

**出现以下情况会主动汇报：**
1. 策略出现瓶颈 (胜率 plateau)
2. 发现新套利模式
3. 市场环境剧变需调整
4. 有显著的优化建议

**优化目标：**
- 不追求高频交易
- 追求高胜率+高盈亏比
- 宁可少做，不做错
