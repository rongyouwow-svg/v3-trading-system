## 🎯 核心保护机制

### 机制 1：进程优先级
```
量化进程：nice -10（高优先级）
其他任务：nice 10（低优先级）

优先级范围：-20（最高）到 19（最低）
```

### 机制 2：自动重启
```
检查频率：每 5 分钟
崩溃检测：进程不存在或端口未监听
自动重启：尝试重启失败进程
日志记录：所有操作记录到日志
```

### 机制 3：资源隔离（可选）
```
CPU 隔离：使用 taskset 绑定 CPU 核心
内存隔离：使用 cgroups 限制内存（如需要）
IO 隔离：使用 ionice 设置 IO 优先级（如需要）
```

## 🛡️ 使用场景

### 场景 1：运行主力研判（大数据量）
```bash
# 确保量化进程高优先级
./process_isolation.sh set

# 以低优先级运行主力研判
./process_isolation.sh run python3 主力研判.py

# 或者直接
nice -n 10 python3 主力研判.py
```

### 场景 2：下载大量数据
```bash
# 限制下载速度，避免占用带宽
# 同时保持量化进程高优先级
./process_isolation.sh set
wget --limit-rate=1m <数据 URL>
```

### 场景 3：运行回测（高 CPU 占用）
```bash
# 回测任务低优先级运行
./process_isolation.sh run python3 backtest.py

# 量化系统不受影响
```

### 场景 4：系统维护
```bash
# 查看当前进程优先级
./process_isolation.sh show

# 检查进程状态
./check_processes.sh
```

## ⚠️ 注意事项

### 1. cron 权限
```
确保 cron 有执行权限：
crontab -l  # 查看当前 cron
```

### 2. 日志轮转
```
日志文件会持续增长，建议定期清理：
# 每月清理一次旧日志
find /home/admin/.openclaw/workspace/quant/ -name "*.log" -mtime +30 -delete
```

### 3. 进程优先级限制
```
普通用户 nice 值范围：0-19
root 用户 nice 值范围：-20-19

如需设置负值（高优先级），可能需要 sudo
```
