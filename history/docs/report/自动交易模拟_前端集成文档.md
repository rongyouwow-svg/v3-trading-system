
API 服务将在 5001 端口运行。

### 方法 2：集成到统一网关
```python

# 注册蓝图
app.register_blueprint(auto_sim_app, url_prefix='/auto_sim')
```

### 方法 3：systemd 服务（推荐）
```bash
# 创建服务文件
sudo nano /etc/systemd/system/auto-sim-api.service
```

```ini
[Unit]
Description=Auto Trading Sim API
After=network.target

[Service]
Type=simple
User=admin
WorkingDirectory=/home/admin/.openclaw/workspace/quant
ExecStart=/usr/bin/python3 /home/admin/.openclaw/workspace/quant/auto_sim_api.py
Restart=always

[Install]
WantedBy=multi-user.target
```

```bash
# 启动服务
sudo systemctl daemon-reload
sudo systemctl enable auto-sim-api
sudo systemctl start auto-sim-api
```

```javascript
    const response = await fetch('/api/strategies/auto_sim/start', {
        method: 'POST',

        body: JSON.stringify({
            symbol: 'ETHUSDT',

    const data = await response.json();
    return data.strategy_state;
}

// 停止策略
async function stopAutoSim() {
    const response = await fetch('/api/strategies/auto_sim/stop', {
        method: 'POST',
        headers: {'Content-Type': 'application/json'},
        body: JSON.stringify({symbol: 'ETHUSDT'})
    });
    const data = await response.json();
    console.log(data);
}
```

## ⚠️ 注意事项

### 1. 端口占用
```
API 服务使用 5001 端口
如被占用，修改 auto_sim_api.py 中的端口号
```

### 2. 策略状态管理
```
策略状态保存在内存中
服务重启后状态丢失
生产环境建议使用 Redis 等持久化存储
```

### 3. 并发处理
```
当前版本不支持多实例并发
同一 symbol 只能运行一个策略实例
如需多实例，修改 simulators 字典结构
```

### 4. 实盘风险
```
⚠️ 此策略仅用于模拟盘测试
⚠️ 实盘前请充分测试
⚠️ 无止损止盈，实盘风险极高
```
