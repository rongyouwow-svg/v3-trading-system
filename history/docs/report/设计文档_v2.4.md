## 🔌 端口设计 ⭐ **核心**

### 端口隔离原则

**核心思想：**
```
✅ 功能隔离 - 不同服务使用不同端口段
✅ 互不冲突 - 端口段不重叠
✅ 便于管理 - 端口用途明确
✅ 扩展性好 - 预留未来服务端口
```

### 端口分配总览

| 服务类型 | 主端口 | 备用端口 | 用途 |
|---------|-------|---------|------|
| **统一网关** | **8080** | 8081-8084 | API 服务 + 策略管理 |
| **HTTP 备用** | **8085** | 8086-8090 | 静态文件服务 |
| 监控系统 | 8091 | 8092-8095 | 监控服务（预留） |
| WebSocket | 8096 | 8097-8100 | 实时推送（预留） |

### 网关服务段：8080-8084

```
8080: 主网关端口 ✅ 当前使用
8081: 备用 1（旧主网关端口）
8082: 备用 2（旧 HTTP 端口）
8083: 备用 3
8084: 备用 4
```

**启动逻辑：**
```python
def find_available_port(cfg):
    ports = [8080, 8081, 8082, 8083, 8084]
    for port in ports:
        if is_port_available(port):
            return port  # 找到第一个可用端口
    return random.randint(8080, 8099)  # 兜底
```

### HTTP 服务段：8085-8090

```
8085: 主 HTTP 端口 ✅ 当前使用
8086: 备用 1
8087: 备用 2
8088: 备用 3
8089: 备用 4
8090: 备用 5
```

**启动逻辑：**
```bash
# start.sh
PORT=8085  # HTTP 服务主端口
```

### 端口隔离优势

**场景 1：网关重启**
```
网关重启
  ↓
检查 8080 → 可用 → 使用 8080 ✅
（只在 8080-8084 范围内切换，不会占用 HTTP 端口）
```

**场景 2：HTTP 服务重启**
```
HTTP 服务重启
  ↓
检查 8085 → 可用 → 使用 8085 ✅
（只在 8085-8090 范围内切换，不会占用网关端口）
```

**关键点：**
```
✅ 网关只在 8080-8084 切换
✅ HTTP 只在 8085-8090 切换
✅ 两个范围完全不重叠
✅ 永远不会互相占用
```


### 保证金占用检查

**代码逻辑：**
```python
def check_margin_usage(symbol, quantity, price, leverage):
    account = client.get_futures_account()
    available_margin = float(account['availableBalance'])
    required_margin = (quantity * price) / leverage
    max_allowed = available_margin * 0.80
    
    if required_margin > max_allowed:
        return False, "保证金占用超过 80%"
    
    return True, "通过"
```
