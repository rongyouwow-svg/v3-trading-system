### 1. 方向逻辑修复原则 🔴

**用户要求：**
```
方向逻辑修复：完全调用策略信号的多空，所以我们的策略必须带多空信号
```

**理解：**
- ❌ 当前：前端根据策略名判断方向（auto_sim=long，其他=short）
- ✅ 应该：策略本身返回多空信号，前端/后端直接使用

**问题：**
- 自动交易模拟策略代码中确实定义了 `action: "BUY"`（开多）
- 但策略启动时，前端传递了 `side: 'short'` 覆盖了
- 现在修复为前端根据策略名判断，但这不是长久之计

**正确方案：**
```python
# 策略引擎返回信号时，应该包含方向
result = strategy_engine.generate_signal(...)
# result 应该包含:
{
    'action': 'OPEN',
    'side': 'long',  # 或 'short'
    'quantity': 0.5,
    ...
}

# 后端直接使用策略返回的方向
side = result.get('side', 'long')
```

**待办：**
- [ ] 检查策略引擎是否返回方向信号
- [ ] 修改后端使用策略返回的方向
- [ ] 前端不再硬编码或判断方向

```javascript
    // ❌ 启动失败（互斥检查失败等）

}
```

**后端逻辑：**

**待确认：**
