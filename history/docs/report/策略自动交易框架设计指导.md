### 2. 网关服务层

**文件：** `gateway.py`

**职责：**
```
✅ API 路由

   - 接收前端请求

   - 接收策略信号
   - 验证信号格式
   - 转换为 API 调用
   - 调用币安客户端

✅ 策略管理
   - 加载策略插件

   - 管理策略生命周期

✅ 日志记录
   - 所有 API 调用日志
   - 策略状态变更日志
   - 错误日志

❌ 不负责
   - 策略逻辑（何时开仓/平仓）
```

## 🔄 数据流详解

### 策略启动流程

```
1. 前端用户操作
   ↓
   选择策略：auto_sim
   选择币种：ETHUSDT
   输入杠杆：2
   输入金额：500
   点击 [启动策略]
   
2. 前端发送请求
   ↓
   POST /api/strategies/start
   Content-Type: application/json
   Body: {
     "strategy": "auto_sim",
     "symbol": "ETHUSDT",
     "leverage": 2,
     "amount": 500
   }
   
3. 网关接收请求
   ↓
   路由：gateway.py → auto_sim_start()
   
4. 风控检查
   ↓
   - 检查 ETHUSDT 是否有其他策略
   - 检查 ETHUSDT 是否有持仓
   - 检查可用资金是否足够
   
5. 创建策略实例
   ↓
   from strategies.auto_sim import AutoSimStrategy
   strategy = AutoSimStrategy(client, "ETHUSDT", 2, 500)
   auto_sim_strategies["ETHUSDT"] = strategy
   
6. 启动策略（异步）
   ↓
   asyncio.run(strategy.start())
   
7. 策略执行逻辑
   ↓
   T+0 秒：emit_signal({type: 'OPEN', side: 'BUY', percentage: 0.5})
   T+20 秒：emit_signal({type: 'ADD', side: 'BUY', percentage: 0.5})
   T+40 秒：emit_signal({type: 'CLOSE', side: 'SELL', percentage: 1.0})
   
8. 网关执行信号
   ↓
   on_signal(signal):
     - 风控检查
     - 计算数量
     - 调用 API
     - 更新状态
     - 记录日志
   
9. 返回结果
   ↓
   {
     "success": true,
     "message": "ETHUSDT 策略已启动"
   }
   
10. 前端更新
    ↓
    轮询 GET /api/strategies/status?symbol=ETHUSDT
    显示最新状态和日志
```

---

### 信号执行流程

```
策略生成信号
  ↓
signal = {
  "type": "OPEN",       # 信号类型
  "symbol": "ETHUSDT",  # 交易对
  "side": "BUY",        # 方向
  "percentage": 0.5     # 仓位百分比
}
  ↓
调用 gateway.execute_signal(signal)
  ↓
网关风控检查
  ↓
if not risk_check(signal):
  return {"success": False, "error": "风控失败"}
  ↓
信号转换为 API 调用
  ↓
quantity = calculate_quantity(signal['percentage'])
  ↓
调用币安 API
  ↓
order = client.place_order(
  symbol=signal['symbol'],
  side=signal['side'],
  type='MARKET',
  quantity=f"{quantity:.4f}"
)
  ↓
更新策略状态
  ↓
strategy.position_size = order['executedQty']
strategy.position_price = order['avgPrice']
  ↓
记录日志
  ↓
strategy.log(f"开仓成功：{order['executedQty']} @ {order['avgPrice']}")
  ↓
返回结果
  ↓
return {"success": True, "order": order}
```

## 🔧 常见问题与修复经验（2026-03-11 更新）

### 问题 1：持仓检查字段名错误 ❌→✅ 已修复

**症状：**
- 启动策略时提示"已有持仓"，但实际没有持仓
- 或者有持仓时提示可以启动

**原因：**
币安 API 返回的持仓数据字段名与代码预期不匹配：
- 代码期望：`positionAmt`
- 实际返回：`size` 和 `side`

**修复方案：**
```python
# 修复前（错误）
if pos['symbol'] == symbol and float(pos.get('positionAmt', 0)) > 0:

# 修复后（正确）
position_size = float(pos.get('size', 0))
position_side = pos.get('side', '')
if pos['symbol'] == symbol and position_size > 0:
    return jsonify({
        'success': False,
        'error': f'{symbol} 已有持仓 ({position_size}), 请先平仓'
    })
```

**修复文件：** `gateway_plugin_routes.py`  
**修复位置：** 
- `plugin_strategy_start()` - 持仓检查
- `GatewayWrapper.execute_signal()` - CLOSE 和 EMERGENCY_CLOSE
- `plugin_strategy_stop()` - 自动平仓

### 问题 2：策略停止时卡死 ❌→✅ 已修复

**症状：**
- 策略停止时显示"正在取消止损单..."
- 然后卡住，没有后续响应

**原因：**
- `get_algo_orders` 可能返回空列表或格式不对
- 没有错误处理，异常导致卡死

**修复方案：**
```python
# 修复前（无保护）
algo_orders = client.get_futures_algo_orders(symbol=symbol, limit=10)
for order in algo_orders.get('orders', []):
    if order.get('algoStatus') == 'NEW':
        algo_id = order.get('algoId')
        result = client.cancel_algo_order(symbol, algo_id)

# 修复后（带保护）
try:
    strategy.log("⚠️ 策略停止：正在取消止损单...")
    algo_orders = client.get_algo_orders(symbol=symbol, limit=10)
    orders = algo_orders.get('orders', [])
    strategy.log(f"📋 找到 {len(orders)} 个 Algo 订单")
    
    cancelled_count = 0
    for order in orders:
        if order.get('algoStatus') == 'NEW':
            algo_id = order.get('algoId')
            result = client.cancel_algo_order(symbol, algo_id)
            strategy.log(f"✅ 取消止损单 {algo_id}: {result.get('success', 'unknown')}")
            cancelled_count += 1
    
    strategy.log(f"✅ 共取消 {cancelled_count} 个止损单")
except Exception as e:
    strategy.log(f"⚠️ 取消止损单失败：{e}")
    # 不阻断后续流程
```

**修复文件：** `gateway_plugin_routes.py`  
**修复位置：** `plugin_strategy_stop()` 函数

### 问题 3：测试网订单不成交 ⚠️ 正常现象

**症状：**
- 订单状态一直是 `NEW`
- `executedQty` 始终为 `0.00`

**原因：**
币安测试网对 MARKET 订单有限制，可能不实际成交。

**解决方案：**
这是测试网的限制，实盘会正常成交。测试时关注：
1. ✅ 订单是否正确发送（有 orderId）
2. ✅ 信号逻辑是否正确
3. ✅ 持仓检查是否正常
4. ✅ 策略流程是否完整

### 问题 4：交易记录未更新 ⚠️ 正常现象

**症状：**
- 策略执行了，但交易记录里没有新记录

**原因：**
交易记录是在订单成交后创建的，如果订单未成交（测试网常见），就不会有交易记录。

**解决方案：**
1. 检查订单状态（`executedQty > 0` 表示成交）
2. 测试网环境下，关注信号发送是否正常
3. 实盘会自动创建交易记录

**验证命令：**
