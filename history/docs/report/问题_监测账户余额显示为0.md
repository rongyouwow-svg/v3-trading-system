# 🐛 监测账户余额显示为 0 的问题

**发现时间：** 2026-03-10 11:25  
**优先级：** ⚠️ 中  
**状态：** 已记录  

---

## 🔍 问题描述

**现象：**
```
监测脚本显示：
✅ 账户余额：$0.00

但实际 API 返回：
{
  "balances": [
    {"asset": "USDT", "free": 4745.05, "locked": 249.43, "total": 4994.48},
    {"asset": "USDC", "total": 5000.0},
    {"asset": "BTC", "total": 0.01}
  ]
}
```

**实际余额：** 约$10,000+  
**显示余额：** $0.00  

---

## 📊 原因分析

**问题 1：API 返回结构不匹配**
```python
# 监测脚本期望的结构：
account = {
    "success": True,
    "balance": 10000  # ← 直接返回总余额
}

# 实际 API 返回的结构：
account = {
    "balances": [  # ← 返回资产列表
        {"asset": "USDT", "total": 4994.48},
        {"asset": "USDC", "total": 5000.0},
        {"asset": "BTC", "total": 0.01}
    ]
}
```

**问题 2：监测脚本解析逻辑错误**
```python
# 当前代码（错误）：
account = get_account_info()
if account and account.get('success'):  # ← API 返回没有 success 字段
    balance = account.get('balance', 0)  # ← API 返回没有 balance 字段
    print(f"✅ 账户余额：${balance:,.2f}")

# 应该改为：
account = get_account_info()
if account and account.get('balances'):
    total_balance = sum(b.get('total', 0) for b in account['balances'])
    print(f"✅ 账户余额：${total_balance:,.2f}")
```

---

## 🛠️ 修复方案

### 方案 1：修改监测脚本（推荐）

**修改 `strategy_monitor.py`：**
```python
def get_account_info(account_type="测试盘"):
    """获取账户信息"""
    try:
        response = requests.get(f"{BASE_URL}/testnet/account", timeout=5)
        data = response.json()
        
        # 正确解析币安 API 返回的数据
        if data and 'balances' in data:
            total = sum(b.get('total', 0) for b in data['balances'])
            return {
                'success': True,
                'balance': total,
                'balances': data['balances']
            }
        return None
    except:
        return None
```
