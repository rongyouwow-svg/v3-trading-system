## 1. 架构概览

### 1.1 系统架构图

```
┌─────────────────────────────────────────────────────────┐
│                     前端 (testnet.html)                  │
│  - 策略列表展示                                          │
│  - 启动/停止策略                                         │
│  - 状态监控                                              │
└────────────────────┬────────────────────────────────────┘
                     │ HTTP/REST API
                     ▼
┌─────────────────────────────────────────────────────────┐
│                   统一网关 (gateway.py)                  │
│  - Flask Web 服务器 (端口 8080)                          │
│  - 策略蓝图路由 (/api/strategies/*)                      │
│  - 币安 API 客户端                                        │
│  - 数据层 (DataFeed)                                     │
│  - K 线生成器 (KlineGenerator)                            │
└────────────────────┬────────────────────────────────────┘
                     │ 动态导入
                     ▼
┌─────────────────────────────────────────────────────────┐
│                策略框架 (strategies/)                    │
│  - loader.py        - 策略加载器（热插拔核心）           │
│  - base_strategy.py - 策略基类                           │
│  - test_strategy.py - 测试策略                           │
│  - *.py             - 用户自定义策略                     │
└─────────────────────────────────────────────────────────┘
```

### 1.2 核心组件

| 组件 | 文件 | 职责 |
|------|------|------|
| **统一网关** | `gateway.py` | 提供 HTTP API，管理策略生命周期 |
| **策略蓝图** | `gateway_plugin_routes.py` | 策略相关路由（/api/strategies/*） |
| **策略加载器** | `strategies/loader.py` | 动态加载策略，支持热插拔 |
| **策略基类** | `strategies/base_strategy.py` | 所有策略的父类，定义接口 |
| **策略注册** | `strategies/__init__.py` | 导出策略列表和加载函数 |

### 1.3 数据流

```
用户操作 → 前端 → API 请求 → 网关 → 策略蓝图 → 策略加载器 → 策略实例
                                         ↓
                                    动态扫描 strategies/ 文件夹
                                         ↓
                                    导入策略类 → 创建实例 → 执行
```

### 3.3 策略示例：双均线策略

```python
from strategies.base_strategy import BaseStrategy

class DualMAStrategy(BaseStrategy):
    """双均线策略 - EMA 金叉做多，死叉做空"""
    
    def __init__(self, gateway, symbol, leverage, amount):
        super().__init__(gateway, symbol, leverage, amount)
        self.fast_period = 12  # 快线周期
        self.slow_period = 26  # 慢线周期
        self.fast_ema = []
        self.slow_ema = []
    
    def on_start(self):
        self.log(f"🚀 双均线策略启动：{self.symbol} ({self.fast_period}/{self.slow_period})")
        return True
    
    def on_stop(self):
        self.log(f"🛑 双均线策略停止：{self.symbol}")
        return True
    
    def on_tick(self, price_data):
        price = price_data['price']
        
        # 更新 EMA
        self.update_ema(price, self.fast_ema, self.fast_period)
        self.update_ema(price, self.slow_ema, self.slow_period)
        
        # 需要足够的数据点
        if len(self.fast_ema) < self.slow_period:
            return None
        
        # 金叉：快线上穿慢线
        if self.fast_ema[-2] <= self.slow_ema[-2] and self.fast_ema[-1] > self.slow_ema[-1]:
            self.log("✨ 金叉信号 - 开多")
            return {'type': 'OPEN', 'side': 'LONG', 'percentage': 1.0, 'stop_loss_pct': 0.05}
        
        # 死叉：快线下穿慢线
        if self.fast_ema[-2] >= self.slow_ema[-2] and self.fast_ema[-1] < self.slow_ema[-1]:
            self.log("💔 死叉信号 - 平仓")
            return {'type': 'CLOSE', 'side': 'LONG', 'percentage': 1.0}
        
        return None
    
    def update_ema(self, price, ema_list, period):
        """更新 EMA 列表"""
        multiplier = 2 / (period + 1)
        if len(ema_list) == 0:
            ema_list.append(price)
        else:
            ema = price * multiplier + ema_list[-1] * (1 - multiplier)
            ema_list.append(ema)
            # 保持列表长度
            if len(ema_list) > period * 2:
                ema_list.pop(0)
```

## 4. 热插拔机制

### 4.1 什么是热插拔？

**热插拔** = 无需重启网关即可添加/修改/删除策略

### 4.2 热插拔流程

```
1. 创建策略文件 → strategies/my_strategy.py
2. 调用刷新 API → POST /api/strategies/refresh
3. 验证策略列表 → GET /api/strategies/list
4. 启动新策略 → POST /api/strategies/start
```

### 4.3 热插拔测试

```bash
# 1. 查看当前策略列表
curl http://localhost:8080/api/strategies/list

# 2. 创建新策略
cat > strategies/simple_strategy.py << 'EOF'
from strategies.base_strategy import BaseStrategy
class SimpleStrategy(BaseStrategy):
    """简单策略"""
    def __init__(self, gateway, symbol, leverage, amount):
        super().__init__(gateway, symbol, leverage, amount)
    def on_start(self): return True
    def on_stop(self): return True
    def on_tick(self, price_data): return None
EOF

# 3. 刷新策略列表
curl -X POST "http://localhost:8080/api/strategies/refresh"

# 4. 验证新策略已加载
curl http://localhost:8080/api/strategies/list | python3 -m json.tool

# 输出应包含 simplestrategy
```

### 4.4 热插拔原理

```python
# loader.py 核心逻辑
class StrategyLoader:
    def reload(self):
        self.strategies = {}
        
        # 扫描 strategies/ 文件夹
        for filename in os.listdir(self.folder):
            if filename.endswith('.py') and not filename.startswith('_'):
                # 动态导入模块
                spec = importlib.util.spec_from_file_location(module_name, module_path)
                module = importlib.util.module_from_spec(spec)
                spec.loader.exec_module(module)
                
                # 找到策略类
                for name in dir(module):
                    obj = getattr(module, name)
                    if is_strategy_class(obj):
                        self.strategies[obj.__name__.lower()] = obj
```

### 4.5 热插拔注意事项

| 操作 | 是否需要刷新 | 说明 |
|------|-------------|------|
| 新增策略文件 | ✅ 需要 | 调用 `/api/strategies/refresh` |
| 修改策略代码 | ✅ 需要 | 调用 `/api/strategies/refresh` |
| 删除策略文件 | ✅ 需要 | 调用 `/api/strategies/refresh` |
| 修改策略参数 | ❌ 不需要 | 参数在启动时传入 |
| 修改基类 | ✅ 需要 | **必须重启网关** |


## 7. 故障排查

### 7.1 常见问题速查表

| 问题 | 可能原因 | 解决方案 |
|------|---------|---------|
| 策略列表为空 | 策略文件未加载 | 调用 `/api/strategies/refresh` |
| 策略启动失败 | 策略 ID 错误 | 检查策略类名（小写） |
| 策略启动失败 | 已有活跃策略 | 先停止现有策略 |
| 热插拔不生效 | 未调用刷新 API | 调用 `/api/strategies/refresh` |
| 递归导入错误 | loader 初始化问题 | 使用 `get_loader()` 函数 |
| 网页无法打开 | 网关崩溃 | 重启网关 `python3 gateway.py` |
| 8080 端口被占用 | 多实例运行 | 检查并停止旧实例 |

### 7.3 递归导入错误

**症状：**
```
❌ 加载策略失败 loader.py: maximum recursion depth exceeded
```

**原因：** `loader.py` 在模块级别调用了 `loader.reload()`

**解决方案：**
```python
# 修复前
loader = StrategyLoader()
loader.reload()

# 修复后
loader = None
def get_loader():
    global loader
    if loader is None:
        loader = StrategyLoader()
    return loader
```

### 7.4 网关崩溃

**症状：** 网页无法打开，API 无响应

**诊断：**
```bash
ps aux | grep gateway.py  # 检查进程是否存在
curl http://localhost:8080/api/health  # 检查是否响应
```

**解决方案：**
```bash
# 重启网关
cd /home/admin/.openclaw/workspace/quant
python3 gateway.py > gateway.log 2>&1 &

# 验证
curl http://localhost:8080/api/health
```

### 7.5 端口冲突

**症状：**
```
Address already in use: 0.0.0.0:8080
```

**原因：** 8080 端口已被占用

**解决方案：**
```bash
# 查找占用端口的进程

# 停止旧进程

# 或使用备份端口（8081/8082/8083）
# 网关会自动尝试备份端口
```

## 8. 经验总结

### 8.1 架构设计经验

#### ✅ 成功经验

1. **统一网关架构**
   - 所有功能通过单一端口（8080）暴露
   - 避免 CORS 问题
   - 简化防火墙配置

2. **蓝图分离**
   - 策略路由独立在 `gateway_plugin_routes.py`
   - 便于维护和扩展
   - 避免主网关文件过于庞大

   - 支持快速迭代开发

4. **延迟初始化**
   - `get_loader()` 函数避免递归导入
   - 按需加载，减少启动时间
   - 避免循环依赖

#### ❌ 失败教训

1. **静态注册表**
   - 最初使用 `AVAILABLE_STRATEGIES` 字典
   - 无法支持热插拔
   - **解决：** 改用 loader 动态扫描

2. **模块级初始化**
   - `loader = StrategyLoader()` 在模块级别执行

   - **解决：** 延迟初始化

3. **相对导入**
   - `from .base_strategy import BaseStrategy`
   - 动态导入时失败
   - **解决：** 使用绝对导入

```python
# 策略引擎新增方法
def _create_stop_loss_order(self, symbol, stop_loss_pct):
    """开仓后自动创建 5% 止损单"""
    position = self.client.get_position(symbol)
    ticker = self.client.get_ticker(symbol)
    
    # 计算止损价格（做多：price*0.95，做空：price*1.05）
    stop_price = round(ticker['price'] * (1 - stop_loss_pct), 2)
    
    # 调用币安 CONDITIONAL API
    self.client.place_algo_order(
        symbol=symbol,
        side='SELL' if position['side'] == 'LONG' else 'BUY',
        quantity=abs(position['size']),
        algo_type='CONDITIONAL',
        trigger_price=stop_price,
        working_type='MARK_PRICE'
    )

def _cancel_stop_loss_orders(self, symbol):
    """策略停止时取消所有止损单"""
    orders = self.client.get_algo_orders(symbol)
    for order in orders:
        if order.get('algoType') == 'CONDITIONAL':
            self.client.cancel_algo_order(symbol, order['algoId'])
```
