# 🐛 自动交易模拟策略方向错误问题

**发现时间：** 2026-03-10 11:26  
**优先级：** 🔴 高  
**状态：** 已记录  

---

## 🔍 问题描述

**现象：**
```
策略代码写的是：开多（BUY）
实际执行的是：开空（SHORT）
```

**策略代码（auto_trading_sim.py）：**
```python
if self.cycle_step == 0:
    # 开多 50%
    signal = {
        "action": "BUY",
        "type": "OPEN",
        ...
    }
```

**实际执行结果：**
```
🎯 活跃策略 (1 个):
  ETHUSDT - auto_sim (SHORT) ← 是空单
```

---

## 📊 原因分析

### 可能原因 1：网关默认方向设置

**gateway.py 第 296 行：**
```python
side = data.get('side', 'short')  # ← 默认值是 short！
```

**问题：**
- 前端如果没有传递 `side` 参数
- 或者传递了但网关没正确处理
- 就会使用默认值 `short`

### 可能原因 2：策略 ID 映射错误

**可能情况：**
- 前端选择"auto_sim"策略
- 但网关映射到了其他策略
- 或者策略参数解析错误

### 可能原因 3：自动交易模拟未正确集成

**问题：**
- 自动交易模拟策略是新加的
- 可能网关中没有对应的处理逻辑
- 前端选择了 auto_sim，但网关用了默认参数

## 🛠️ 修复方案

### 方案 1：修改网关默认值

**修改 gateway.py：**
```python
# 当前（问题）：
side = data.get('side', 'short')  # ← 默认空头

# 修改为：
side = data.get('side', 'long')  # ← 默认多头
# 或者
side = data.get('side')  # ← 不设置默认值，必须前端传递
```

### 方案 2：自动交易模拟特殊处理

**在网关中添加特殊逻辑：**
```python
if strategy_name == 'auto_sim':
    # 自动交易模拟策略固定为多头
    side = 'long'
    # 或者根据策略内部状态决定
```

### 方案 3：前端明确传递方向

**修改前端代码：**
```javascript
// 启动策略时明确传递方向
startTrading() {
  const params = {
    strategy: this.strategy,
    side: this.strategy === 'auto_sim' ? 'long' : 'short',
    symbol: this.symbol,
    ...
  }
}
```
