## 1. 系统概述

### 1.1 系统定位

大王量化交易系统 v3.0 是一个**模块化、高可用、易扩展**的币安 U 本位合约自动量化交易系统。

### 1.2 核心目标

- ✅ **100% 年化收益** - 通过优化策略和仓位管理实现
- ✅ **高可用性** - 多进程解耦，单点故障不影响全局
- ✅ **热插拔** - 策略独立运行，网关重启不影响策略
- ✅ **易维护** - 模块化设计，清晰的职责划分
- ✅ **易扩展** - 插件系统，支持多交易所、多策略

### 1.3 支持功能

| 功能类别 | 具体功能 |
|---------|---------|
| **交易功能** | U 本位合约、市价单、限价单、止损单、止盈单 |
| **策略功能** | 突破策略、RSI 反转策略、网格策略（未来） |
| **风控功能** | 仓位监控、资金监控、异常检测、分级告警 |
| **管理功能** | 策略管理、订单管理、交易记录、绩效分析 |
| **扩展功能** | 插件系统、多交易所支持（未来）、Web Dashboard |

## 2. 设计理念

### 2.1 核心设计原则

```
│                                                             │
│  3. 高可用 (High Availability)                              │
│     • 多进程解耦                                             │
│     • 故障隔离                                               │
│     • 自动重启                                               │
│                                                             │
│  4. 易扩展 (Extensible)                                     │
│     • 插件系统                                               │
```

## 4. 核心业务流程

### 4.1 策略启动流程

```
用户 → API 网关 → 策略引擎 → 执行引擎 → 交易所

1. 用户发送 POST /strategy/start
2. 网关验证参数（余额、杠杆）
3. 策略引擎创建策略实例
4. 启动信号计算线程（热插拔）
5. 执行引擎开仓（MARKET BUY）
6. 交易所返回订单确认
7. 执行引擎创建止损单（STOP_MARKET）
8. 交易所返回止损单确认
9. 持久化状态到 plugin_strategies.json 和数据库
10. 返回结果给前端
```

```
   │         │          │          │────────>│
   │         │          │          │ 止损单确认│
   │         │          │          │<────────│
   │         │          │ 持久化状态 │         │
   │         │          │ • plugin_strategies.json
   │         │          │ • SQLite 数据库
   │ 返回结果 │          │          │         │
   │<────────│          │          │         │
```

## 6. 模块详细设计

### 6.1 策略引擎 (Strategy Core)

**职责**: 策略管理、信号生成、回测验证

**核心模块**:
- `StrategyManager` - 策略管理器（热插拔）
- `SignalGenerator` - 信号生成器
- `Backtester` - 回测引擎

**关键特性**:
- ✅ 支持多策略同时运行
- ✅ 策略独立线程，持续信号计算
- ✅ stop_flag 安全停止机制
- ✅ 策略状态持久化

### 6.2 执行引擎 (Execution Core)

**职责**: 订单管理、止损单执行、成交监听

**核心模块**:
- `OrderManager` - 订单管理器
- `StopLossExecutor` - 止损单执行器
- `FillMonitor` - 成交监听器

**关键特性**:
- ✅ 订单生命周期管理（状态机）
- ✅ 止损单防重机制
- ✅ 动态精度获取
- ✅ 重试机制（3 次，指数退避）

### 6.3 风控引擎 (Risk Core)

**职责**: 仓位监控、资金监控、异常检测、告警管理

**核心模块**:
- `PositionMonitor` - 仓位监控
- `CapitalMonitor` - 资金监控
- `AnomalyDetector` - 异常检测
- `AlertManager` - 告警管理

**关键特性**:
- ✅ 实时仓位跟踪
- ✅ 风险敞口计算
- ✅ 异常分类处理
- ✅ 多渠道告警（Telegram/钉钉/邮件）

### 6.4 行情数据层 (Market Data Layer)

**职责**: K 线管理、Ticker 缓存、数据补全、质量校验

**核心模块**:
- `KlineManager` - K 线管理器
- `TickerCache` - Ticker 缓存
- `DataFetcher` - 数据补全器
- `QualityChecker` - 质量校验器

**关键特性**:
- ✅ 多周期 K 线缓存（1m/5m/15m/1h/4h）
- ✅ WebSocket 断线补全
- ✅ 数据质量校验
- ✅ 避免频繁 API 调用

### 6.5 资金管理引擎 (Capital Engine)

**职责**: 仓位计算、PnL 计算、手续费统计

**核心模块**:
- `PositionCalculator` - 仓位计算器
- `PnLCalculator` - PnL 计算器
- `FeeTracker` - 手续费统计

**关键特性**:
- ✅ 开仓大小计算（固定比例/凯利公式）
- ✅ 加仓比例管理（20%→40%→60%）
- ✅ 已实现/未实现 PnL 计算
- ✅ 手续费累计统计

### 6.6 状态同步层 (State Sync Layer)

**职责**: 状态快照、冲突解决、恢复管理

**核心模块**:
- `StateSnapshot` - 状态快照
- `ConflictResolver` - 冲突解决
- `RecoveryManager` - 恢复管理

**关键特性**:
- ✅ 定期全量同步（每 5 分钟）
- ✅ 事件驱动增量同步
- ✅ 版本号检测冲突
- ✅ 以交易所数据为准修复

### 6.7 异常处理引擎 (Exception Engine)

**职责**: 异常分类、重试机制、恢复策略

**核心模块**:
- `ExceptionClassifier` - 异常分类
- `RetryHandler` - 重试处理
- `RecoveryStrategy` - 恢复策略

**关键特性**:
- ✅ 异常分类（网络/业务/系统）
- ✅ 自动重试（指数退避）
- ✅ 降级处理
- ✅ 状态修复

## 7. 数据库设计

### 7.1 表结构

```sql
-- ==================== 策略表 ====================
CREATE TABLE strategies (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    symbol TEXT NOT NULL UNIQUE,          -- 交易对
    strategy_id TEXT NOT NULL,             -- 策略 ID
    strategy_name TEXT NOT NULL,           -- 策略名称
    side TEXT NOT NULL,                    -- LONG/SHORT
    leverage INTEGER DEFAULT 1,            -- 杠杆
    amount REAL DEFAULT 0,                 -- 保证金
    status TEXT DEFAULT 'running',         -- running/stopped/paused
    start_time DATETIME,                   -- 启动时间
    stop_time DATETIME,                    -- 停止时间
    entry_price REAL,                      -- 入场价
    current_price REAL,                    -- 当前价
    position_size REAL,                    -- 持仓数量
    pnl REAL DEFAULT 0,                    -- 盈亏
    pnl_pct REAL DEFAULT 0,                -- 盈亏率
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- ==================== 订单表 ====================
CREATE TABLE orders (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    order_id TEXT UNIQUE,                  -- 交易所订单 ID
    strategy_id INTEGER,                   -- 关联策略
    symbol TEXT NOT NULL,                  -- 交易对
    side TEXT NOT NULL,                    -- BUY/SELL
    type TEXT NOT NULL,                    -- MARKET/LIMIT/STOP_MARKET
    quantity REAL NOT NULL,                -- 数量
    price REAL,                            -- 价格
    avg_price REAL,                        -- 成交均价
    filled_quantity REAL DEFAULT 0,        -- 已成交数量
    status TEXT DEFAULT 'PENDING',         -- PENDING/OPEN/FILLED/CANCELED/REJECTED
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (strategy_id) REFERENCES strategies(id)
);

-- ==================== 止损单表 ====================
CREATE TABLE stop_orders (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    algo_id TEXT UNIQUE,                   -- 交易所止损单 ID
    strategy_id INTEGER,                   -- 关联策略
    symbol TEXT NOT NULL,                  -- 交易对
    side TEXT NOT NULL,                    -- BUY/SELL
    trigger_price REAL NOT NULL,           -- 触发价
    quantity REAL NOT NULL,                -- 数量
    status TEXT DEFAULT 'WAIT_TO_TRIGGER', -- WAIT_TO_TRIGGER/TRIGGERED/CANCELED
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    triggered_at DATETIME,                 -- 触发时间
    FOREIGN KEY (strategy_id) REFERENCES strategies(id)
);

-- ==================== 成交表 ====================
CREATE TABLE trades (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    trade_id TEXT UNIQUE,                  -- 交易所成交 ID
    order_id TEXT,                         -- 关联订单
    strategy_id INTEGER,                   -- 关联策略
    symbol TEXT NOT NULL,                  -- 交易对
    side TEXT NOT NULL,                    -- BUY/SELL
    quantity REAL NOT NULL,                -- 数量
    price REAL NOT NULL,                   -- 价格
    commission REAL DEFAULT 0,             -- 手续费
    commission_asset TEXT DEFAULT 'USDT',  -- 手续费币种
    realized_pnl REAL DEFAULT 0,           -- 已实现盈亏
    trade_time DATETIME,                   -- 成交时间
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (order_id) REFERENCES orders(order_id),
    FOREIGN KEY (strategy_id) REFERENCES strategies(id)
);

-- ==================== 状态快照表 ====================
CREATE TABLE state_snapshots (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    snapshot_type TEXT NOT NULL,           -- strategy/order/position
    snapshot_data TEXT NOT NULL,           -- JSON 数据
    version INTEGER NOT NULL,              -- 版本号
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- ==================== 索引 ====================
CREATE INDEX idx_strategies_symbol ON strategies(symbol);
CREATE INDEX idx_strategies_status ON strategies(status);
CREATE INDEX idx_orders_symbol ON orders(symbol);
CREATE INDEX idx_orders_status ON orders(status);
CREATE INDEX idx_stop_orders_symbol ON stop_orders(symbol);
CREATE INDEX idx_trades_symbol ON trades(symbol);
CREATE INDEX idx_trades_strategy ON trades(strategy_id);
```

## 9. 技术选型

### 9.1 核心技术栈

| 组件 | 技术 | 理由 |
|------|------|------|
| **API 框架** | FastAPI | 高性能、异步、自动文档 |

| **进程管理** | systemd/supervisor | 成熟稳定、自动重启 |
| **日志** | structlog + JSON | 结构化、易分析 |
| **监控** | Prometheus + Grafana | 标准方案、可视化强 |
| **Web 框架** | Vue3 + Element Plus | 现代、组件丰富 |
| **部署** | Docker + docker-compose | 标准化、易部署 |
