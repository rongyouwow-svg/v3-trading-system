## 📋 必须监控的核心点

### 1. 策略层监控

| 监控点 | 检查内容 | 告警条件 | 优先级 |
|--------|----------|----------|--------|
| **进程运行** | 策略进程是否存活 | status != "running" | 🔴 P0 |
| **信号产生** | signals_sent 是否增长 | 30 分钟无增长 | 🟡 P1 |
| **信号执行** | signals_executed 是否等于 sent | executed < sent | 🔴 P0 |

**为什么监控这些**:
- ✅ 进程运行 = 策略正常工作
- ✅ 信号增长 = 策略在分析市场
- ✅ 信号执行 = 开单流程正常

**不监控什么**:
- ❌ RSI 值（每个策略不同）
- ❌ 具体指标（策略内部逻辑）
- ❌ 信号条件（策略自己决定）

---

### 2. 执行层监控

| 监控点 | 检查内容 | 告警条件 | 优先级 |
|--------|----------|----------|--------|
| **开单执行** | 有信号是否有持仓 | signals_sent>0 但无持仓 | 🟡 P1 |
| **止损设置** | 有持仓是否有止损 | 有持仓但无止损单 | 🔴 P0 |
| **杠杆一致** | 实际杠杆是否匹配 | leverage != config | 🟢 P2 |

**为什么监控这些**:
- ✅ 开单执行 = 信号→开单流程正常
- ✅ 止损设置 = 风险控制正常
- ✅ 杠杆一致 = 仓位控制正常

---

### 3. 持仓层监控

| 监控点 | 检查内容 | 告警条件 | 优先级 |
|--------|----------|----------|--------|
| **持仓存在** | 是否有意外持仓 | 无信号但有持仓 | 🟡 P1 |
| **持仓盈亏** | 亏损是否过大 | pnl < -50 USDT | 🟡 P1 |
| **止损触发** | 止损是否执行 | 止损触发但持仓仍在 | 🔴 P0 |

## 🚨 告警处理流程

### 收到告警后

```
1. 查看告警内容
   ↓
2. 定位问题类型
   ├─ 策略异常 → 检查进程
   ├─ 信号未执行 → 检查 API
   ├─ 无止损单 → 立即设置
   └─ 持仓异常 → 手动干预
   ↓
3. 查看相关日志
   ├─ 策略日志
   ├─ API 日志
   └─ 监控日志
   ↓
4. 执行修复
   ├─ 重启策略
   ├─ 手动设置止损
   └─ 手动平仓
   ↓
5. 验证修复
   └─ 再次检查状态
```

## 🔧 告警修复手册

### 告警 1: 策略未运行

**症状**: `status != "running"`

**处理**:
```bash
# 1. 检查进程
ps aux | grep strategy

# 2. 查看日志
tail -50 logs/*_strategy.log

# 3. 重启策略
supervisorctl restart quant-strategy-eth

# 4. 验证
supervisorctl status
```

---

### 告警 2: 信号未执行

**症状**: `signals_sent > signals_executed`

**处理**:
```bash
# 1. 检查 API 状态
curl http://localhost:3000/api/health

# 2. 查看策略日志
tail -50 logs/*_strategy.log

# 3. 检查持仓
curl http://localhost:3000/api/binance/positions

# 4. 可能需要手动开单
```

---

### 告警 3: 有持仓无止损

**症状**: `有持仓但无止损单`

**处理**:
```bash
# 1. 立即设置止损
curl -X POST http://localhost:3000/api/binance/stop-loss \
  -H "Content-Type: application/json" \
  -d '{"symbol":"ETHUSDT","trigger_price":2146.99,"quantity":0.017}'

# 2. 验证止损单
curl http://localhost:3000/api/binance/stop-loss

# 3. 检查策略代码
grep "create_stop_loss" strategies/*.py
```

---

### 告警 4: 持仓亏损过大

**症状**: `pnl < -50 USDT`

**处理**:
```bash
# 1. 查看持仓详情
curl http://localhost:3000/api/binance/positions | python3 -m json.tool

# 2. 检查止损单是否设置
curl http://localhost:3000/api/binance/stop-loss

# 3. 决定是否手动平仓
# 如果止损正常，等待触发
# 如果止损异常，立即设置或平仓
```

## 🎯 监控原则

### 要监控的

1. ✅ **进程是否运行** - 最基础
2. ✅ **信号是否产生** - 策略在工作
3. ✅ **信号是否执行** - 流程正常
4. ✅ **止损是否设置** - 风险控制
5. ✅ **持仓是否异常** - 意外情况

### 不监控的

1. ❌ **RSI 值** - 每个策略不同
2. ❌ **具体指标** - 策略内部逻辑
3. ❌ **余额** - 容易误报
4. ❌ **策略逻辑** - 策略自己决定
