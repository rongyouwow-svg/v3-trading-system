**时间**: 2026-03-19 16:40  
**问题**: ETH 止损单没有跟进立即开启

---

## 🔍 问题深度分析

### 问题现象

```
✅ 策略有持仓：0.15 ETH @ 2168.58
❌ 但没有止损单
```

### 根本原因

**V3 系统工作流程缺陷**:

```
1. 策略启动 → 2. 同步持仓 → 3. 发现有持仓 → 4. 但没有创建止损单
                                              ↓
                                    代码逻辑缺失！
```

**具体原因**:

1. **策略启动时发现已有持仓**（之前开的）
2. **代码只在开仓后调用 create_stop_loss()**
3. **启动时发现已有持仓，没有调用 create_stop_loss()**
4. **结果：有持仓但无止损**

---

### 第 2 步：修复止损价格计算逻辑

**问题**: 使用入场价计算止损，但已亏损时会立即触发

**修复前**:
```python
stop_price = self.entry_price * (1 - self.stop_loss_pct)
# 入场价 2168.58，止损 0.2% → 2164.25
# 但当前价已经 2160，会立即触发！
```

**修复后**:
```python
# 获取当前价格
current_price = get_current_price()

# 如果已亏损，使用当前价格计算止损（避免立即触发）
if current_price < self.entry_price:
    stop_price = current_price * (1 - self.stop_loss_pct)
    print(f"⚠️ 当前已亏损，使用当前价计算止损")
else:
    stop_price = self.entry_price * (1 - self.stop_loss_pct)
    print(f"✅ 使用入场价计算止损")
```

## 📋 如何发现这个问题？

### 监测系统应该具备的能力

| 检查项 | 当前 | 应该有 | 状态 |
|--------|------|--------|------|
| 进程是否运行 | ✅ 监控 | ✅ 监控 | ✅ |
| 持仓是否存在 | ❌ 不监控 | ✅ 应该监控 | ❌ |
| 止损单是否存在 | ❌ 不监控 | ✅ 应该监控 | ❌ |
| 持仓和止损单匹配 | ❌ 不监控 | ✅ 应该监控 | ❌ |

### 改进方案

**添加持仓 - 止损单匹配检查**:

```python
def check_position_stop_loss_match():
    """检查持仓和止损单是否匹配"""
    positions = get_positions()  # 获取所有持仓
    stop_orders = get_stop_orders()  # 获取所有止损单
    
    for pos in positions:
        if pos['size'] > 0:  # 有持仓
            # 查找对应的止损单
            has_stop = any(o['symbol'] == pos['symbol'] for o in stop_orders)
            if not has_stop:
                alert(f"⚠️ {pos['symbol']} 有持仓但无止损单！")
                create_stop_loss(pos)  # 自动创建
```

**集成到监控脚本**:
```bash
# monitor.sh 添加检查
check_stop_loss_match() {
    python3 scripts/check_stop_loss_match.py
}

# 每 5 分钟检查一次
if [ $((cycle % 5)) -eq 0 ]; then
    check_stop_loss_match
fi
```
